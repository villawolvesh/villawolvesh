## ⭐ Featured Project

### Automated 3-Statement Financial Dashboard
**[▶ Live Dashboard](https://villawolvesh.github.io/automated-3-statement-dashboard/)** · [Source & methodology](https://github.com/villawolvesh/automated-3-statement-dashboard)

An interactive financial dashboard built on **real, official SEC EDGAR data** — not sample or simulated numbers. It links the three financial statements (Income, Balance Sheet, Cash Flow), computes ratios, common-size and variance analysis, and lets you compare any period (annual or quarterly, YoY or QoQ).

- **Real data, automated:** a Python pipeline pulls filings straight from the SEC, lands them in SQL, and rebuilds the dashboard on its own — weekly, with zero manual work.
- **Auditable by design:** every figure is computed in visible SQL and passes an automated integrity check (the three statements must reconcile, or it won't publish).
- **Reads the story, not just the numbers:** built-in notes flag and explain real anomalies — e.g. why Apple's FY2024 net income fell despite higher revenue (a one-time EU tax charge), or why its ROE runs above 150% (share buybacks).

> Built with Python (pandas) · SQLite + SQL · Plotly · GitHub Actions

## Why these choices (how it works in a real team)

This isn't a one-off chart — it's built the way a finance team actually needs to operate:

- **Official source over a loose file** → numbers come from an auditable source of truth, not a spreadsheet edited by hand.
- **Visible SQL** → anyone on the team can trace exactly where each figure comes from.
- **A shareable link, not a heavy file** → send a URL anyone opens in a browser, no software to install — everyone sees the same, always-current version.
- **Automated refresh** → financial numbers evolve (preliminary → reported → occasionally restated). The pipeline always reflects the latest filed figures and flags when a number was revised, so the team is never working off stale or hand-updated data.
